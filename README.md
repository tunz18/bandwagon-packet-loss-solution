## 丢包率高是什么，先把概念搞清楚

**丢包（Packet Loss）**指的是你从本地发出去的数据包，没有成功到达目标服务器——或者到了但没有收到回应。ping 测试返回的丢包率，就是这种"有去无回"的比例。

几种常见情况要分开看：

- **5% 以内**：基本正常，偶发网络抖动，不用管
- **10%–30%**：体感明显，网页加载慢，SSH 卡顿；需要查原因
- **50% 以上**：非常严重，几乎不能正常使用
- **100% 丢包 + 国内 ping 不通但国外能通**：这基本就是 IP 被封了，不是普通的网络问题

这个区分很重要。**IP 被封**和**线路丢包**是两回事，处理方式完全不同，后面会分开讲。

**用搬瓦工官方 IP 检测工具**。登录 KiwiVM 控制面板，找到 IP 检测入口（或者直接在 KiwiVM 里查看 IP 状态），如果显示 "IP BLOCKED"，就确认是 IP 被封。

## 第二步：排查常见的线路丢包原因

如果确认不是 IP 被封，丢包率高还有几种可能：

**1. 晚高峰网络拥堵**

这是最常见的原因之一。很多人平时用着没问题，晚上 8 点到 11 点开始卡，这基本是线路带宽饱和的信号。

搬瓦工的基础 KVM 套餐走的是普通国际线路（163 骨干网），这条线平时还行，但晚高峰承载压力大，丢包明显。用下来的感受是：下午两三点基本没问题，到了晚上八九点就开始抽风。

**2. 机房线路本身质量问题**

不同机房的线路质量差别很大。搬瓦工旗下的 DC2、DC4、DC8 等普通机房，走的是质量相对一般的直连线路；而 DC6 CN2 GIA-E、DC9 CN2 GIA 这些机房，走的是电信 CN2 GIA 专线，丢包控制明显好得多。

CN2 GIA 是什么？简单说就是中国电信单独维护的高端商用骨干网（AS4809），和普通 163 骨干网（AS4134）是两套系统。CN2 GIA 线路全程走专网，不和大众流量抢带宽，晚高峰表现稳很多。

**3. 当前所在机房临时抖动或维护**

搬瓦工偶尔会对机房做网络调整或设备维护，这种时候某个机房可能出现短暂的丢包。去搬瓦工官网查一下 Global Status（全球状态页）可以看当前有没有已知问题。

**4. 本地网络运营商的出口拥堵**

有时候问题不在搬瓦工这边，而是你自己的 ISP 出口堵了。切换手机热点测一下，如果丢包消失，就是你本地宽带的问题。

## 第三步：几种实用的排查步骤

1. **先用全球 ping 工具测**：比如 ping.pe 或者 itdog.cn，从多个地区同时 ping 你的 VPS IP，看看是国内节点丢包还是全球都丢包
2. **对比不同时段**：白天和晚高峰各测一次，如果只有晚上丢包，基本就是线路拥堵
3. **用 traceroute 看路由**：在 Windows 跑 `tracert [你的IP]`，在 Mac/Linux 跑 `traceroute [你的IP]`，看看在哪一跳开始出现丢包
4. **切换机房测试**：在 KiwiVM 控制面板里可以一键切换机房，如果换了机房就好了，说明是原来那个机房的问题
